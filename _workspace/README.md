# Workspace Handoffs

이 디렉토리는 하네스 중간 산출물과 핸드오프를 저장한다.

## 파일 규칙

- 파일명 형식: `{phase}_{agent}_{artifact}.md`
- 예시:
  - `00_lead_project_brief.md`
  - `01_design_wireframe.md`
  - `02_api_contracts.md`
  - `02_frontend_handoff.md`
  - `03_qa_report.md`
  - `04_release_report.md`

## 사용 원칙

- 긴 맥락은 채팅보다 `_workspace/` 파일에 남긴다.
- 워커는 자신의 산출물 경로를 명시적으로 받는다.
- 다음 단계가 필요로 하는 정보만 남기고, 원문 로그 복붙은 줄인다.

## 기본 단계

1. 브리프 정리
2. 와이어프레임 및 컴포넌트 맵
3. API 계약과 백엔드 핸드오프
4. 프론트엔드 핸드오프
5. QA 보고
6. 릴리스 보고
